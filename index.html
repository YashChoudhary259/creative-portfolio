<about>

Not just clean code. Code that feels good to use.

I'm a front-end developer who cares less about which framework is trending and more about the feeling an interface gives you the second it loads. HTML, CSS, and JavaScript are the toolkit — React is where I reach when state gets complicated. But the real work happens in the details: the 200ms it takes a button to respond, the easing on a hover state, the difference between a page that works and one that feels considered.

Most recently that's meant building a quiz app fast enough to feel like a game, and a handful of layouts obsessed with getting spacing and rhythm exactly right. Next up: going deeper on full-stack, so the things I design don't stop at the front end.
